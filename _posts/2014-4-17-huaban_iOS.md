<h3 style="color:#268bd2;">1、花瓣IOS客户端用户分析</h3>
<h4 style="color:#268bd2;">1．1花瓣网用户群分析</h4>
<h4 style="color:#268bd2;">1.1.1核心用户</h4>

**①设计师、摄影师、艺术家：**收集灵感、善于发现美<br>
**角色模型：**小A，女，设计师，25岁，工业设计研究生，即将踏入社会，在接设计项目和制作作品集与个人网站时都会用花瓣收集灵感，拥有几十个画板，会按类别与主题分画板。
**需求：**<br>
素材积累：收集整理自己感兴趣的设计案例和教程等内容，用于学习和交流（like站酷）；<br>
自我认同：发现和分享美，证明自己是一位热爱工作的有为青年，灵感枯竭时也会无聊地找乐子；<br>
同行与大众认同：有时上传原创作品，在被收集或者评论时获得成就感，希望上首页或被认证，甚至被客户看中；

**②学生、白领、主妇：**空闲时间较多、喜欢浏览与分享，女性为主<br>
**角色模型：**小B，女，学生，23岁，浙江大学光电系的女生，平时略有文艺，喜欢电影，喜欢各种美好的图片，看到就会收集，拥有各种分类的十几个画板，在想要为社交网络配图或者无聊的时候都会浏览。<br>
**需求：**<br>
悠闲地打发时间：无聊的时候随手打开，漫无目的的浏览，看到美图就会心动，关注一些有趣的画板和人物跟踪，对各类美好的图片都有收藏欲（like豆瓣线上活动）；<Br>
购物：通过花瓣礼品这个比较文艺，经过筛选的导购平台来收藏自己喜欢的物品，在某种契机下会买下来（like豆瓣东西、堆糖、暖岛）；<br>
社交需求：花瓣内部的社交圈（生人圈，以兴趣为导向），和分享到外部吸引关注的途径；<br>
作为工具：电影、拍照POSE、娱乐需求（花瓣娱乐）等；<br>
私密性：不便于公开的图片能得到私密保护；<br>

**③有收藏整理癖的人：**有某种特殊喜好、或收藏整理癖的人<br>
**角色模型：**
小C，男，公司职员，极喜欢汉服，致力于复兴汉服文化，主动，收集网上的各类汉服相关图片，参与了本地的一个汉服活动小组，会定期进行汉服相关的活动，如摄影等，会将活动内容上传致各图片平台及花瓣共享。<br>
**需求：**<br>
寻找同类：共享和获得兴趣归属，认识志同道合的人；<br>
便利的收藏方式：能从多种途径收集，容易管理和归类，能够继承画板和编辑；<br>
关注度：活动共享的平台，希望更多的 人了解、喜欢和加入自己的活动；

<h4 style="color:#268bd2;">1.1.2非核心用户</h4>
主要群体为：商务人士、工科男，主要用花瓣为某项工作收集图片或截图，当做一次性工具。<br>
**角色模型：**
小D，男，28岁，博士，计算机深度用户，喜欢用chrome浏览器，因为快，简洁，一般不安装插件。在某特殊项目需要收集大量网页图片数据或者截图的时候在知乎寻觅到花瓣插件，使用后觉得满意，但为了减少占用资源插件用后即卸。<br>
**需求：**<br>
收集便利：插件好用不崩溃；<br>
网页访问无障碍：速度快，没有无法进入或者被封号的风险；<br>
私密性：不便于公开的图片能得到私密保护；

<h4 style="color:#268bd2;">1.2花瓣网iOS客户端的用户分析</h4>
<h4 style="color:#268bd2;">1.2.1 iOS设备持有者特征</h4>
根据易观[国际的数据](http://www.enfodesk.com/SMinisite/newinfo/topicsdetail-info_id-116.html 、http://www.enfodesk.com/SMinisite/newinfo/topicsdetail-info_id-118.html )可以知道iOS设备持有者最多的群体主要属性有：25-30岁、中高收入、女性偏多。

<h4 style="color:#268bd2;">1.2.2花瓣iOS客户端用户</h4>
综合花瓣网的用户和iOS设备持有者的特点，结合花瓣作为一个图片类应用主要的使用场景（室内、稳定WiFi、光线较好、闲暇）可以得到其IOS客户端的用户主要为花瓣网用户中的「学生、白领、主妇」，以及部分「设计师、摄影师、艺术家」（因为他们对图片的质量有更高的需求）和部分「有收藏整理癖的人」，其典型用户模型就是花瓣IOS客户端的用户模型。

<h3 style="color:#268bd2;">2 花瓣iOS客户端价值分析</h3>
<h4 style="color:#268bd2;">2.1用户价值</h4>
<h4 style="color:#268bd2;">2.1.1信息的有序组织</h4>
每个人将图片整理到画板的过程是一种将信息有序化的过程，这样的组织提供给用户一个类似「笔记」的记录分类功能，并且这种过程可以在传播中完成，提高组织效率，这种由用户重新组织的兴趣内容本身也会吸引更多人的关注。

<h4 style="color:#268bd2;">2.1.2基于兴趣图谱的生人社交圈、表达个性</h4>
花瓣是一个基于图片的兴趣类社交平台，用户能够通过图片的信息和他人对画板的整合方式判断一个人的兴趣与个性，同时也是展示自己兴趣与个性的平台，一方面能够通过这样的方式找到同类形成基于兴趣的社交圈，形成良性互动，同时也通过个性的表达与展示（甚至上主页）获取满足感和成就感。（基于此也许可以创建兴趣小组的形式？）

<h4 style="color:#268bd2;">2.1.3信息过滤与筛选</h4>
兴趣与内容匹配。在人们挚爱的兴趣上以及在人生的重要时刻，往往希望大范围地寻找参考信息，这时候花瓣能够他们一定的帮助。在他们需要设计灵感、想了解一种风格、寻找送朋友的礼物、休假目的地、美食、结婚礼服、家居与装修风格时，都会想要有一个能够轻松展示「这个就是我想要的」的图片的平台，而不是绞尽脑汁地在各种网站进行各种关键字搜索。

<h4 style="color:#268bd2;">2.1.4发现和拥有美</h4>
就像有些淘宝中一些有名的「旅行」店铺，里面的衣服大多是平常人不会穿的，但很多客户买的是「美的需求」，看着图片很美就想拥有，女性有一种天然的对美的渴望。在花瓣可以发现美、收集美、甚至拥有美的东西。

<h4 style="color:#268bd2;">2.2商业价值</h4>
<h4 style="color:#268bd2;">2.2.1电商导购</h4>
显然信息筛选的需求中有很大一部分是对想要购买或者需要购买的物品的需求，在不影响用户进行其他操作的前提下通过导购的方式能够为网站带来一定的商业价值。
但导购不仅仅是流量的引导，也有一部分来源于「创造需求」，很多时候用户并没有对某一件东西的刚性需求，只是在浏览过程中被商品呈现的美感和「气质」吸引，在正好能够购买的情况下冲动购买。

<h4 style="color:#268bd2;">2.2.2基于兴趣的广告</h4>
类似豆瓣广告的模式，根据用户的兴趣精准地投放广告（eg:某个「3D打印图片展示」的画板集合或者活动是由某家3D打印机制造商冠名或者编辑的，可以是每张图片下小小的水印，也可以是其中一张图片的宣传）。但广告形式需要与整体图片风格和质量匹配，不能为了商业价值影响用户体验，这样用户的流失势必使商业价值减弱。

<h4 style="color:#268bd2;">2.2.3用户的力量</h4>
作为一个UGC平台，用户的力量不可小觑，如果能够建立分享驱动力和激励的机制，在用户质量和数量都达到一定水平的情况下，也许会通过用户本身的活动本迸发出意想不到的商业价值。

<h3 style="color:#268bd2;">3基于用户定位和价值分析的花瓣iOS客户端功能设想</h3>
<h4 style="color:#268bd2;">3.1浏览关注的人和画板的功能</h4>
作为一个网站的客户端，最需要保证的功能自然是与网页功能的协同和延续，在web上收集的图片、关注的人和花瓣都能在手机上看到。
在现有的功能基础上也许可以增加「离线画板」和「私密画板」的功能。

<h4 style="color:#268bd2;">3.1.1离线画板</h4>
**场景设想：**每次拍照时苦恼不知道如何摆POSE，于是在花瓣收集了一个POSE画板，满心欢喜。心血来潮出门拍照的时候发现打开花瓣图片缓冲不出来，刷图要很多流量，后悔没有保存，回去打开WiFi想要下载，下了几张之后觉得一张张保存太麻烦，还是算了吧。<br>
**功能设想：**在「我的画板」中设置一个默认的「离线画板」，将图片保存到离线画板时默认WIFI时下载到本地，用户也可自定义某个画板离线，离线的画板内图片不会在清除缓存时被清除，但可以自定义选择清除。“离线画板”在web上可选择一键下载画板内全部图片到本地文件夹。

<h4 style="color:#268bd2;">3.1.2私密画板</h4>
**场景设想：**想要做一个「如何瘦腿」的画板，但是之前的画板都是摄影和设计类的，并且因为图片质量比较高有较多的关注着，觉得这个画板影响了自己的整体形象而且内容本身有点私密，不太想别人知道，苦于花瓣没有「私密画板」，只能选择下载收集或者寻找其他平台。（或者做某个项目时想收集一些数据，部分数据是网页信息的截图、自己做的表格截图等私人内容，不希望共享给大家。）

**功能设想：**在创建画板时可以选择创建「私密画板」或「共享画板」，由于共享画板会有用户关注，选择「共享」的画板将不允许改变为「私密」，而「私密」的画板则可以转化为「共享」。

<h4 style="color:#268bd2;">3.2上传图片的功能</h4>
即现存的上传功能。也许可以将这个功能隐藏在画板内，因为花瓣手机客户端最主要的功能还是以浏览和分享为主，而且并非熟人社区，大部分分享的图片不是手机相册里的内容，上传的频率并不会特别高。对自己某个画板的上传渠道在自己的画板里，对协作画板的上传渠道则放在协作画板主页，若要上传到新建画板则需首先新建画板。可以考虑支持多图上传。

<h4 style="color:#268bd2;">3.3随机浏览「我想看」的图片的功能</h4>
用户在空闲或者无聊寻找灵感的时候希望在手机里能够看到并不是自己关注的内容，但也不希望翻看很久都是不喜欢的图片进而失去兴趣。所以在随机浏览时可以设定一些方便用户筛选的分类。也可以开启「热门」、「分类」、「类genius」、「类shuffle」的功能。

<h4 style="color:#268bd2;">3.3.1热门</h4>
可以是热门采集等内容，因为是基于用户的选择，在用户人数和质量保证的前提下可以通过用户本身的行为发现大众最关心的东西，也许是某个电影或电视的相关图片，也许是某个事故的报道和相关内容（eg：马航失事时关于历史上的空难（比较消极，但是会更容易被分享到其他平台）、危难中的感人瞬间、对于生命珍贵的感慨等），也许是热门科技事件（eg:科学菠萝奖、诺贝尔奖的获奖情况）等等，不过在用户本身不足以支撑比较主流的信息引导的情况下可以通过编辑发布（但是最好能营造出是用户发布的假象，就是用喂养过的「马甲」，能够带动图片整体质量）。

「热门」的展现形式：只是一个“热门”（或者很犀利的其他名字）的标签，但是点进去之后就有上述的比较有趣的内容。但同类热门事件的图片不能不断反复出现甚至刷屏的现象（eg:top50都是「都教授」的情况会引起部分人的极度反感和对网站品味的质疑）、比较丰富多彩并且总数不能过多（eg：每天的热门提供100张图片，一方面防止用户觉得「这哪里是热门啊明明就是所有东西的随机排列」，另一方面类似「饥饿营销」的效果，如「NEXTDAY」不能提供明天的内容这样的限定，会培养用户每天打开App的习惯。）

<h4 style="color:#268bd2;">3.3.2分类</h4>
「分类」的类别数量可以减少，类目概括，因为在选择的时候看到满屏的文字会让部分用户心生倦意，或者在某个类别中再细分。（在分类中是否有必要更明显地暗示「商品」？）

<h4 style="color:#268bd2;">3.3.3类genius</h4>
也就是现存的「猜你喜欢」的栏目，可以通过对用户兴趣DNA的分析提供相关的内容，并且最好能够自动刷新（每次打开或者每天自动刷新一遍，或者只是顺序打乱重新排列也是好的，这样会给用户一种「你一直在不停地给我提供服务，想主动提供给我优质内容」的感觉）。

<h4 style="color:#268bd2;">3.3.4类shuffle</h4>
就是「穿越门」，有时候就是懒得思考和选择，按一下这个按钮能够随机出现各种内容，这一部分对图片的内容没有特别的要求，但最好能够是同类图片里质量较高的部分，呈现新的世界，同时也有点儿类似「摇一摇」的惊喜。

<h4 style="color:#268bd2;">3.4专题板块</h4>
IOS客户端所针对的部分群体，如学生、白领、主妇，会无目的或只是为了打发时间寻找和关注一些事件或图片，这样的用户手机内安装的能够提供资讯和各类信息的应用会比较多，假如没有特别的吸引和需求不会主动打开花瓣客户端。所以可以提供一个类似「每日话题」或者「线上活动」的专题来培养用户每天打开客户端的习惯（类似「知乎」的「知乎日报」）。<br>
专题的内容可以是WEB上的「花瓣设计」中的「专题」、「视觉新青年」、「周刊」或者微信订阅号中推送的文章，也可以是「线上活动」的形式（like豆瓣线上活动），每个专题起一个特别的名字，这样会增加用户评论、回复等互动的频率，形成更浓的社交氛围。

「线上活动」设想：花瓣娱乐提供的协作功能允许多个用户协同参与组织信息，大大提高基于兴趣发现同类信息和个人的可能性，可以尝试着用在其他模块，基于某个花瓣发起的主题或者用户自主发起的话题进行投稿、甚至可以某商家进行有奖活动（eg：「一个」发起「晒我与《去你家玩好吗》的合影赠签名书」活动）。投稿的方式是在活动页面点击「提交」可以选择自己画板内或者本地相册内的内容上传参与。可以以画板封面图片平铺性下滑加日期（或时间轴）的形式呈现。（or拟物？）

<h4 style="color:#268bd2;">3.5个人主页</h4>
个人主页是展示自己的一个舞台，所以需要能够给用户一定的「自定义」的空间，如更换头像和昵称、更换背景页面等。但也不能提供太多自定义功能，总有些用户会无节制地把美好的东西弄得有点糟糕。

<h3 style="color:#268bd2;">4 一点思考</h3>
<h4 style="color:#268bd2;">4.1和其他客户端相比的区别</h4>
<h4 style="color:#268bd2;">4.1.1与图片类社交应用区别</h4>
如：图钉、啪啪、instagram、微信朋友圈等。

这些社区都是熟人社区，围绕现实生活，图片大多描述私人生活，吸引很多评论与关注但较小的分享价值，用户能够通过与朋友分享身边的事物获取存在感和共同语言。花瓣与之相比更侧重「兴趣」属性，基于兴趣的生人兴趣小组的形式，在这一点上定位更接近豆瓣。因此与这一类应用其实并没有特别大的冲突，可以共存。

<h4 style="color:#268bd2;">4.1.2与导购类图片应用的区别</h4>
如：蘑菇街、美丽说、明星衣橱、逛、凡客达人、迷尚网、麦糖网等。

这些导购应用风格各异，或者小清新或者走高端路线或者以搭配和实惠来吸引人，但均为纯导购应用，其用户定位与花瓣IOS客户端虽然有部分重合，但花瓣提供的物品有自己的性格，并且除了导购之外能够实现更多功能，在有机会接触的情况下能够吸引部分原来使用导购类应用的用户。

<h4 style="color:#268bd2;">4.1.3 与图片收集类网站的区别</h4>
如：堆糖、我喜欢、发现啦、嘀咕网、hello设计网等，以上网站目前并非都有客户端，假设以后有。

这一类应用与花瓣的定位最为相似，竞争最大。个人认为想要取胜最重要的首先是内容的高质，无论是图片所包含的内容或者图片分辨率、画板的分类明确性都会影响用户体验。可以通过主动编辑和激励机制加以改善；其次是基于兴趣分析提供针对性内容的能力，用户总会希望有一个「更懂我」的产品，满足其天生的渴望被理解的需求；第三是交互和视觉上一些细节的设计，更美更方便；第四是社交成分的养成，增加粘性和离开的代价。（如前述成立小组、线上活动、协同画板、「更有趣」的一些小创意等方式）。

<h4 style="color:#268bd2;">4.1.4与设计类应用的区别</h4>
网站：如站酷、Dribbble、优设网<br>
轻博客类网站：点点、qing、lofter

<h4 style="color:#268bd2;">4.1.4与豆瓣社区的区别</h4>
其实从某种意义上而言花瓣客户端与豆瓣社区的用户群体非常相似，尤其与其中「豆瓣东西」和「豆瓣线上活动」两个部分，但单从图片的角度而言花瓣有很大的优势（由于豆瓣东西和豆瓣线上活动目前为止均没有官方客户端，这里分析的是假设或者网页版的情况）。首先豆瓣在IOS端拥有至少10个客户端（包括非官方），每一个客户端提供单一的功能，这在一定程度上增加用户使用的门槛，有些用户会选择只下载其中的几个，对于似乎不是特别有用的豆瓣相册或者东西会选择不下载或者试用后删除；其次豆瓣线上活动和豆瓣东西虽然有豆瓣原有的庞大用户群的支持，但在功能目前为止加起来也没有花瓣强大，用户存在选择功能更完善的应用的可能；第三豆瓣近年来因为部分板块用户质量的下降，流失了部分高质量的用户，在还没有更好的选择情况下选择花瓣作为其中图片板块的替代也不是不可能。总的说起来豆瓣也许值得花瓣学习和借鉴，也可能吸引一部分用户转移。（去豆瓣搞几个线上活动推花瓣？）

<h4 style="color:#268bd2;">4.2关于图片内容的一些想法</h4>
觉得对于一个UGC平台来说，最重要的还是内容。内容要保质保量可能还是需要编辑和马甲们的努力。想到一部分礼品类的内容整合可以参考淘宝店铺中特别火的一些文艺类的店铺，因为首先受众比较相似，其次很多人买这些店铺的商品只是出于美的追求，而使用概率很小，足以见里面的图片和风格对这个群体的吸引力。另外也许可以拓展一下男性用户，增加一些类似（新闻、武器、历史、战争、数据？）等之类的内容。
